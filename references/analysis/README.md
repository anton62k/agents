# Analysis References

Analysis references help agents define what should be built and how to know it
is done.

## Practices

- requirements elicitation and validation;
- current and desired behavior analysis;
- stakeholder, context, and constraint discovery;
- task decomposition and dependency ordering;
- acceptance criteria and open-question tracking;
- user, system, and process flow description.

## Used By

- `analyst` as primary owner;
- `architect` when requirements affect boundaries or contracts;
- `reviewer` when checking whether a diff satisfies the approved task spec;
- `orchestrator` at route-selection depth only.

## Source Anchors

- IIBA BABOK: business analysis knowledge areas and techniques.
- ISO/IEC/IEEE 29148: requirements engineering processes and work products.
- User Story Mapping: shared understanding and user-centered decomposition.
